# 产品需求文档：AI 智能刷题小程序

## 1. 产品概述

- **产品名称**：题刷刷 — AI 智能刷题助手
- **一句话摘要**：利用 AI 技术为求职者提供个性化、高效的刷题体验
- **问题陈述**：求职面试准备过程中，候选人面临以下痛点：
  - 题库质量参差不齐，难以找到针对性的题目
  - 刷题过程枯燥，缺乏反馈和进步可视化
  - 无法根据个人水平动态调整难度
  - 碎片时间利用率低

- **目标用户**：
  - 主要用户：正在准备技术面试的应届生和职场人（22-35 岁）
  - 次要用户：需要定期考核的行业从业者（如金融、法律持证人员）

## 4. 技术方案

- **架构**：小程序端 + 云函数后端 + AI 服务
- **前端**：微信小程序（原生开发）
- **后端**：Node.js 云函数 / Python FastAPI
- **AI**：接入 LLM API（DeepSeek / GPT）实现智能出题和解析
- **存储**：云数据库（用户数据、题目、答题记录）
- **关键依赖**：AI 模型 API 的稳定性和延迟、题库数据质量

### 数据模型核心实体

```
User (id, name, avatar, target_role, level)
Question (id, content, options, answer, knowledge_tags, difficulty)
Answer (id, user_id, question_id, answer, is_correct, created_at)
WrongBook (id, user_id, question_id, tag, created_at)
StudyStats (id, user_id, total_count, correct_count, daily_data)
```

### 风险与缓解

| 风险 | 缓解方案 |
|------|---------|
| AI 生成题目质量不稳定 | 加人工审核 + 用户反馈机制 |
| API 调用延迟影响体验 | 流式输出 + 题目预缓存 |
| 用户留存率低 | 每日推送 + 学习排行 + 打卡 |

## 5. 成功指标

- **北极星指标**：用户每周完成刷题数 ≥ 50 题
- **首要指标**：
  - 次日留存率 ≥ 40%
  - 7 日留存率 ≥ 20%
- **次要指标**：
  - 平均每次使用时长 ≥ 10 分钟
  - 题目满意度评分 ≥ 4.0/5.0
- **反指标**：
  - AI 生成题目的错误率（目标 ≤ 5%）

## 6. 开放问题

1. 题目版权问题：AI 生成的题目是否涉及版权风险？需要法务评估。
2. 免费 vs 付费：MVP 阶段完全免费还是设置免费额度？
3. 题库冷启动：初期没有用户提交真题，题库如何快速建立？
